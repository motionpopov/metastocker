# MetaStocker Blog Design Guidelines

This document provides ai-instructions and design guidelines for generating new blog articles under the `/blog/` directory on the MetaStocker website.

## General Principles

1. **No External Frameworks**: The site relies heavily on a lightweight, custom CSS structure (`style.css`) augmented by partial inline Tailwind classes (`<script src="https://cdn.tailwindcss.com"></script>`) exclusively for structure/layout padding.
2. **Minimalist & Clean**: The design favors a minimal UI scheme, utilizing explicit CSS variables for theming (supporting both light and an eventual dark mode based on media queries).

---

## File Structure

Every new article must reside in the `/blog/` folder as a static `.html` file.
Duplicate `article-template.html` to begin, and inject your content inside the `<article class="article-content ...">` tag.

---

## Typography & Elements (`.article-content` scope)

The styling is explicitly handled via the injected `<style>` tag in the `<head>` of the blog templates. Ensure your raw HTML output sticks to standardized Markdown-equivalent tags inside the `<article>` wrapper.

### Headings
- Use `<h2>` for main section titles. They map to `24px` with a `-0.02em` tracking and `700` weight.
- Use `<h3>` for sub-sections. They map to `20px`.
*Note: Do not add custom classes to headings inside `.article-content`; standard `<h2>`/`<h3>` is sufficient.*

### Paragraphs & Lists
- Use standard `<p>`, `<ul>`, `<ol>`, and `<li>` tags. They automatically adopt the `16px` font size, `1.7` line height, and standard margin/padding.
- Strong text is rendered using `<strong>` (bolded at `600` weight).

### Blockquotes
- Used for callouts or notable quotes. Generated by wrapping text in `<blockquote>`. They feature a subtle left-border and italic text using `var(--subtle)`.

### Code & Formatting
- Code blocks must use `<pre><code>`. The background uses `var(--muted)` with appropriate internal padding and layout properties.
- Inline code uses `<code>`.

### Images & Figures
- For images with captions, wrap them in a `<figure>` tag.
- Use the `<img>` tag for the actual image. The template automatically applies rounding, borders, and shadows.
- Use `<figcaption>` for the descriptive text below the image.

Example:
```html
<figure>
  <img src="path/to/image.jpg" alt="Description of image">
  <figcaption>This is the caption for the image.</figcaption>
</figure>
```

## MetaStocker UI Toolkit (CSS Variables & Classes)

If you need to construct custom UI elements outside the standard `.article-content` (like new cards or custom layouts), use these tokens.

### Colors (CSS Variables)
- `--bg`: Main page background.
- `--card`: Background for raised elements (like the article body).
- `--muted`: Subtle background tone (often used for code blocks or hovers).
- `--text`: Primary readable text.
- `--subtle`: Secondary, lower contrast text (hints, subtitles).
- `--border`: Standard interface borders.
- `--ring`: Primary brand accent color (often blue).
- `--accent`: High contrast call-to-action color (Dark/Black).
- `--accent-contrast`: Opposite of accent.

### Interactive Components

**Buttons**
- `.btn.primary`: Main CTA button (Dark background, light text).
- `.btn.ghost`: Secondary button with transparent background, borders on hover.

**Containers & Cards**
- `.card.pad`: Standard container with border, soft shadow, and `20px` internal padding. Add `hover:bg-muted/30` or similar Tailwind utilities for interactivity.

**Text Utilities**
- `.headline`: Heavy, compressed tracking text.
- `.subtitle`: `14px` auxiliary text with `--subtle` color.
- `.hint`: `12px` descriptive text.
- `.badge`: Small pill-shaped tag usually bordered, containing short metadata or tags.

## Checklist for AI Generators

- [ ] Copy the HTML boilerplate exactly from `/blog/article-template.html`.
- [ ] Update the `<title>`, `<meta description>`, and `<link rel="canonical">` for SEO.
- [ ] Correctly set the Category Badge text, Date, and Read Time in the article header.
- [ ] Implement article body using standard HTML structure (`h2`, `p`, `ul`, `blockquote`) inside `<article class="article-content">`.
- [ ] Check if the CTA segment and FAQ correctly align with the overarching context of the piece. (Expand the FAQ if relevant to the article context).
- [ ] DO NOT add external CSS links other than `../style.css` and the Tailwind CDN.
